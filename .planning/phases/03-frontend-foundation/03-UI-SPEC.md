# Phase 3 — UI Design Contract

> Visual and interaction contract for the Frontend Foundation phase (FE-01–FE-08).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Aesthetic target: dark, data-dense Bloomberg-style trading terminal. Desktop-first.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS + custom dark theme — shadcn deliberately not used per CONTEXT D-05/D-08; data-dense terminal built from primitives, not a component library) |
| Preset | not applicable |
| Component library | none (raw HTML/JSX + Tailwind; charts via TradingView Lightweight Charts) |
| Icon library | none required in Phase 3 (connection status is a CSS dot, not an icon). If an icon is needed, use `lucide-react` (tree-shakeable, matches terminal weight) |
| Font | JetBrains Mono for all numeric/tabular data (price, change%, cash, portfolio value); system sans stack (`ui-sans-serif, system-ui, ...`) for labels/headers. Monospace tabular figures keep price columns aligned — required for terminal feel (CONTEXT Discretion, resolved). |

**Notes on sources:** Color tokens locked by PLAN.md §2 and REQUIREMENTS FE-03. Charting library, state split, layout, and flash mechanism locked by CONTEXT.md D-01–D-10. Font choice resolved from CONTEXT "Claude's Discretion" toward monospace for the terminal aesthetic.

## Spacing Scale

8-point scale. Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Cell inner padding, gap between sparkline and price column, dot-to-label gap |
| sm | 8px | Watchlist row vertical padding (compact rows), header item gaps |
| md | 16px | Default padding inside the header bar and watchlist panel container |
| lg | 24px | Gap between header and watchlist panel; panel outer padding |
| xl | 32px | Major layout column gaps |
| 2xl | 48px | Reserved (panel separation in later phases) |
| 3xl | 64px | Reserved (page-level spacing in later phases) |

**Exceptions:**
- Watchlist rows are compact (target ~28–32px row height) to fit all 10 tickers without a scrollbar (CONTEXT D-08, FE-05). Row height uses `8px` vertical padding on `14px`/line-height-1.2 text — this is an intentional density exception to the default `md` padding, not a new spacing token.
- Selected-row left accent bar is exactly `2px` wide (CONTEXT D-10) — a fixed border width, not a spacing token.

## Typography

Three roles declared (terminal density needs few sizes). Two weights only.

| Role | Size | Weight | Line Height | Family | Usage |
|------|------|--------|-------------|--------|-------|
| Display | 20px | 600 (semibold) | 1.2 | JetBrains Mono | Header portfolio total value (the single largest live number) |
| Body / Data | 14px | 400 (regular) | 1.2 | JetBrains Mono | Watchlist price, change%, ticker symbol, cash balance — all tabular numeric data |
| Label | 12px | 600 (semibold) | 1.2 | system sans | Column headers (Symbol / Price / Change% / Sparkline), connection-status label, section captions |

**Weights:** exactly two — 400 (regular) for data, 600 (semibold) for the display value and labels. No other weights.

**Line height:** 1.2 throughout — data-dense terminal layouts use tight leading, not the 1.5 body default. Justification: no long-form prose in this phase; every row is a single line of tabular data.

**Tabular figures:** Enable `font-variant-numeric: tabular-nums` on all numeric columns so prices do not shift horizontally as digits change during flashing.

## Color

60/30/10 split applied to the dark terminal theme. All hex values locked by PLAN.md §2 / FE-03 — do not substitute.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d1117` | Primary app background, watchlist panel base surface |
| Secondary (30%) | `#1a1a2e` | Header bar background, selected-row subtle background lift, panel insets |
| Accent (10%) | `#ecad0a` | See reserved-for list below — strictly limited |
| Blue (secondary accent) | `#209dd7` | Sparkline line color, interactive/hover affordance on rows, neutral chart strokes |
| Purple | `#753991` | Submit/primary-action buttons (no buttons in Phase 3 — reserved for Phase 4 trade/chat submit) |
| Up / profit | `#22c55e` (green) | Price flash-up background tint, positive change% text, uptick direction |
| Down / loss | `#ef4444` (red) | Price flash-down background tint, negative change% text, downtick direction |
| Text primary | `#e6edf3` | Default text on dark surfaces (no pure white) |
| Text muted | `#8b949e` | Column header labels, secondary/flat text, borders (no pure black) |
| Border | `#30363d` | Muted gray row dividers and panel borders |

**Accent (`#ecad0a`) reserved for — explicit list:**
- The 2px left accent bar on the currently selected watchlist row (CONTEXT D-10)
- The connection-status dot **only when needed for the connected state? No** — the connected dot is green, not accent. Accent is **not** used for status dots.

Accent yellow `#ecad0a` is reserved exclusively for the selected-ticker indicator in this phase. It is never used for general text, borders, hover states, or the connection dot.

**Connection status dot colors (FE-02, CONTEXT specifics):**
- Connected → green `#22c55e`
- Reconnecting (EventSource CONNECTING) → yellow-amber `#ecad0a` is acceptable here as a semantic "warning" reuse, OR a distinct amber `#f59e0b`. **Decision: use `#f59e0b` (amber) for reconnecting** so accent yellow stays uniquely the selection signal. Avoids overloading `#ecad0a`.
- Disconnected (EventSource ERROR after retries) → red `#ef4444`

**Semantic up/down:** Green/red are the only semantic pair beyond accent. They carry profit/loss and uptick/downtick meaning everywhere — flash animation, change% text color, and (Phase 4) heatmap.

**Flash tint guidance:** Flash background uses a low-opacity green/red (e.g. `rgba(34,197,94,0.25)` / `rgba(239,68,68,0.25)`) on the price cell, transitioning back to transparent over ~500ms. The base text color of the price cell stays readable throughout.

## Interaction Contract

This phase is interaction-heavy (live streaming) but light on copy. Interaction states are first-class contract items.

| Interaction | Trigger | Visual Contract |
|-------------|---------|-----------------|
| Price flash up | SSE update with `direction === "up"` | Add `.flash-up` to price cell: green background tint, fades to transparent over 500ms via `transition: background-color`. Remove class after 500ms `setTimeout`. (FE-06, CONTEXT D-07) |
| Price flash down | SSE update with `direction === "down"` | Add `.flash-down` to price cell: red background tint, same 500ms fade. (FE-06, CONTEXT D-07) |
| Price flat | `direction === "flat"` | No flash; price updates silently. |
| Change% color | On each update | Green text if change% ≥ 0, red text if < 0. Muted text `#8b949e` if exactly flat. |
| Sparkline growth | Each SSE tick since page load | Append point to that ticker's Lightweight Charts mini-instance; line color `#209dd7` (blue). Starts empty and fills progressively — no historical backfill. (FE-07, CONTEXT D-06) |
| Row select | Click anywhere on a watchlist row | Apply 2px `#ecad0a` left bar + subtly lighter `#1a1a2e` row background. Only one row selected at a time. Selection feeds the Phase-4 main chart. (FE-08, CONTEXT D-10) |
| Row hover | Pointer over a row | Subtle background lift toward `#1a1a2e` (lighter than base, weaker than selected). Cursor `pointer`. |
| Connection status | EventSource readyState changes | Header dot recolors: green (OPEN), amber (CONNECTING), red (ERROR after retries). Driven by Zustand `connectionStatus`. (FE-02) |
| Header live values | SSE-derived portfolio recompute | Portfolio total value (20px display) and cash balance (14px data) update live without layout shift (tabular-nums). |

## Copywriting Contract

Phase 3 has minimal text — no forms or destructive actions yet. The required copy:

| Element | Copy |
|---------|------|
| Primary CTA | None in Phase 3 (no buttons; first CTA is the Phase-4 trade "Buy" / "Sell"). Mark as N/A. |
| Column headers | `Symbol` · `Price` · `Change %` · (sparkline column header left blank or `Chart`) |
| Header labels | `Portfolio` (precedes total value) · `Cash` (precedes cash balance) |
| Connection status (connected) | Dot only; optional adjacent label `Live` |
| Connection status (reconnecting) | Dot only; optional adjacent label `Reconnecting…` |
| Connection status (disconnected) | Dot only; optional adjacent label `Disconnected` |
| Empty state heading | `No prices yet` |
| Empty state body | `Waiting for the live market feed…` — shown in the watchlist panel before the first SSE event arrives (sparklines and prices populate progressively). |
| Error state | `Live feed lost — reconnecting automatically.` Shown when `connectionStatus === "disconnected"`; no user action required since EventSource auto-retries. No retry button needed. |
| Destructive confirmation | None in Phase 3 (watchlist remove / trades are Phase 4). Mark as N/A. |

**Tone:** Terse, factual, terminal-style. No exclamation marks. Numbers and tickers carry the message; prose stays out of the way.
