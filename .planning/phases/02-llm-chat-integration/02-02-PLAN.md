<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client -> POST /api/chat | User-supplied message string crosses here; arbitrary text |
| LLM response -> auto-executor | Structured JSON from OpenRouter/Cerebras contains tickers, sides, quantities; treated as untrusted input |
| chat_messages table -> LLM prompt | Prior conversation rows are included in next LLM prompt; could carry injected content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | LLM-supplied ticker in execute_trade_on_conn | mitigate | Ticker is `.strip().upper()` normalized before any SQL; all SQL uses parameterized queries — no string formatting of LLM values |
| T-02-06 | Tampering | LLM-supplied side/quantity in execute_trade_on_conn | mitigate | side validated against {"buy","sell"} inside helper; quantity must be > 0; both return failure dict on invalid input rather than proceeding |
| T-02-07 | Tampering | Prompt injection via user message | accept | Stakes are simulated money; all trades still pass cash/shares validation; LLM cannot skip the validation path — execute_trade_on_conn is the only code path for trades |
| T-02-08 | Information Disclosure | OPENROUTER_API_KEY in logs or error responses | mitigate | litellm reads key from env; never log messages[] at INFO level; error response is generic {"error": "LLM unavailable"} with no key content |
| T-02-09 | DoS | Blocking litellm.completion in async route | mitigate | `await asyncio.to_thread(completion, ...)` keeps event loop responsive during inference; SSE stream unblocked |
| T-02-10 | DoS | Unbounded conversation history | mitigate | LIMIT 20 on history query (D-04) caps prompt size regardless of how many messages exist |
| T-02-11 | DoS | Malformed LLM JSON crashing route | mitigate | model_validate_json failure is caught in except Exception; returns HTTP 500 {"error": "LLM unavailable"} — no unhandled exception |
| T-02-12 | Tampering | SQL injection via LLM-supplied content in chat_messages INSERT | mitigate | All INSERTs use parameterized queries (?, ?, ...) — LLM message content is never string-formatted into SQL |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan — litellm already locked at 1.87.1 in uv.lock |
</threat_model>
