<objective>
Refactor the existing trade execution and watchlist mutation logic in portfolio.py and watchlist.py to expose connection-level helper functions that the upcoming chat auto-executor can call without making circular HTTP requests.

Purpose: CHAT-03 requires that trades auto-executed by the LLM go through the identical validation path as manual trades. The only way to guarantee this is to extract the DB mutation logic into a shared helper that both the HTTP route and the chat route call. Same principle applies to watchlist changes (CHAT-04).

Output: Two new module-level functions — `execute_trade_on_conn` in portfolio.py and `apply_watchlist_change_on_conn` in watchlist.py — with the existing HTTP routes refactored to be thin wrappers calling them.
</objective>
